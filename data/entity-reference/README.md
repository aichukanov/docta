# Справочный контент для страниц услуг и анализов

Черновое хранилище текстов для контент-слоя из [[service-reference-content]] (`prd/service-reference-content/`). Позже — миграция в SQL, аналогично `medical_service_tariffs` в [[fzocg]].

## Файлы

- `lab-tests.json` — справки для `lab_tests` (`pages/labtests/[labTestSlug]/`)
- `medical-services.json` — справки для `medical_services` (`pages/services/[serviceSlug]/`)

## Схема (1 элемент массива)

```json
{
  "slug": "complete-urinalysis",
  "entity_type": "lab_test",
  "translations": {
    "ru": {
      "what": "1 фраза — что это",
      "how": "как физически проводится, сколько занимает",
      "when": "типичные показания",
      "prep": "условия подготовки",
      "abnormal": "нейтрально — «может быть связано с ...», обязательно с отсылкой к врачу"
    }
  }
}
```

`slug` — должен совпадать с `lab_tests.slug` / `medical_services.slug` в БД (связь по слагу, не по id — id в JSON не хранится).

`translations` — по мере перевода добавляются ключи `en`, `sr`, `sr_cyrl`, `de`, `tr` с тем же набором из 5 полей. Отсутствие локали в `translations` = перевод ещё не готов, не показывать/фолбэк на `ru` — решается на этапе вывода.

**`sr_cyrl` не переводится отдельно** — генерируется механической транслитерацией из `sr` (латиница→кириллица, тот же принцип, что в `common/serbian-transliteration.ts`). Но международные лабораторные обозначения нужно защищать от транслитерации явным списком (см. `PROTECTED_LATIN_TOKENS` в скрипте генерации) — иначе получается мусор («HbA1c» → «ХбА1ц») или омографы с обычными сербскими словами («PSA» → «ПСА», совпадает с род. падежом слова «пас»/собака). Устоявшиеся сербские кириллические аббревиатуры (ХИВ, ЕКГ, ХОБП, МР, ОРЛ, АСТ, АЛТ, ЛДЛ, ХДЛ) транслитерируются нормально — в список защиты не входят. При добавлении новых карточек с незнакомыми латинскими аббревиатурами — проверять результат sr_cyrl глазами, а не доверять транслитерации вслепую.

Дисклеймер («информация не заменяет консультацию врача...») в данных не хранится — единая i18n-строка на уровне шаблона страницы, показывается для каждой карточки.

## YMYL-правила (см. подробно в PRD)

Без числовых референсов, без диагностических формулировок («X означает болезнь Y»), только «может быть связано с» + отсылка к врачу.
