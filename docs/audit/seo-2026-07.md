# SEO-аудит docta.me — июль 2026

## Ситуация на момент аудита

Сайт упёрся в ~1000 показов/день при ~54k страниц в индексе (~0,02 показа/страницу/день).

**Цифры GSC (снимок ~2026-07-04, окно апрель–июнь):** ~54k страниц не в индексе (33 672 crawled-not-indexed, 11 418 noindex, 7 622 redirect, 341 «Google выбрал другой canonical», 830 404, 77 5xx, 16 soft-404), ~54k в индексе.

**Инфляция URL:** реальных сущностей ~15–18k (1 208 врачей, ~500 клиник, услуги, анализы, 2 523 лекарства, фасетные комбинации), но ×6 языков через `?lang=` + комбинаторика фильтров дают ~108k известных Google URL. Индекс заполнен страницами, которые Google считает не стоящими показа: главная проблема — не «не индексируется», а «в индексе, но не показывается».

**Ссылочный профиль:** 1 081 внешняя ссылка, из них 972 — собственный busticket4.me (сквозная), остальные ~100 унаследованы через 301 с купленных брошенных доменов клиник (bolnica-meljine, domzdravljahercegnovi, ordinacijanatal, malbaski — анкоры это выдают). Заработанных ссылок ≈ 0.

**Внутренняя перелинковка:** топ по входящим — /privacy (40k), footer-страницы. На фильтрованные city×specialty страницы HTML-ссылок нет вообще — специальности упомянуты в doctor/specialties.vue как текст, крошки врача не поднимаются до уровня город/специальность.

## Что уже реализовано в коде (проверено на 2026-07)

- hreflang + canonical в app.vue (`getCanonicalUrl` — единая точка)
- sitemap с alternates и порогами (`server/common/sitemap/`)
- 301 с числовых ID и старых слагов (`server/middleware/redirect-routes.ts`)
- noindex на пустых списках (`list-page.vue`)
- Пагинация листингов написана (`LIST_PAGE_SIZE`, `?page=`, rel prev/next в `list-page.vue`), но по состоянию на 2026-07-19 всё ещё не закоммичена (+149 строк в рабочей копии) — в проде `/doctors` отдаёт все 1 208 профилей одной страницей.

Важное следствие: техническая база (hreflang, canonical, 301, noindex, sitemap) в порядке, и цифры GSC это подтверждают. Дальнейшие чисто технические правки дадут мало — потолок в другом.

## Ключевой вывод

svad.net того же владельца с той же архитектурой даёт 10k показов/день → дело не в технике, а в YMYL-пороге (медицина) + нулевом доверии домена + thin-страницах. Товарному агрегатору Google верит «по умолчанию», медицинскому — только при явных сигналах доверия.

**Калибровка ожиданий:** сербоязычный медицинский спрос в стране на 620k человек конечен; кратный рост лежит в ru/en-сегменте (релоканты, экспаты, медтуризм), где контент уже есть, но доверия домена пока нет.

## Детализация пунктов (чтобы брать в работу по одному)

### 1. Перелинковка

Сейчас весь внутренний вес уходит в footer (топ-получатель — /privacy, 40k ссылок), а на коммерческие фасетные страницы не ведёт ни одной HTML-ссылки — они живут только в sitemap, без веса и анкоров.

- `components/doctor/specialties.vue` — специальности выводятся текстом; сделать ссылками на `/doctors?specialtyIds=N` (+ город врача: `&cityIds=M`).
- `pages/doctors/[doctorSlug]/index.vue` — крошки сейчас «Главная → Врачи → Имя» (и в schema.org, и видимые AppBreadcrumbs); добавить уровень «Стоматологи Подгорицы» (`/doctors?specialtyIds=N&cityIds=M`).
- Link-хабы на листингах: внизу `/doctors` блок «Стоматологи в Будве / Баре / Никшиче…» и «Другие специальности в {городе}» — переиспользовать пороги из `server/common/sitemap/filters/doctors.ts`, чтобы не ссылаться на пустые комбинации.
- Аналогично для клиник (тип×город), услуг и анализов (категория×город) — комбинации уже вычисляются для sitemap, тот же источник данных.
- То же для карточек клиник/услуг/анализов: крошки с уровнем город/категория.

### 2. Пагинация

Код готов в рабочей копии `components/list-page.vue` (+149 строк: `LIST_PAGE_SIZE`, `?page=`, rel prev/next). Осталось: закоммитить, задеплоить. Пока не задеплоено, `/doctors` рендерит все 1 208 профилей — любой фасетный срез выглядит строгим подмножеством родителя, Google склеивает. Учесть: страницы `page≥2` — `noindex,follow` или canonical без page (решить при реализации); в пагинаторе нужны настоящие `<a href>` для передачи веса, не только кнопки.

### 3. Сжатие индексируемой поверхности

Цель — атаковать 33 672 crawled-not-indexed и 11 418 noindex: меньше мусорных URL, больше веса оставшимся.

- **Сначала данные:** выгрузить из GSC выборки «просканировано, но не проиндексировано» и «noindex» — понять состав (языковые варианты? лекарства? пустые фасеты?). От этого зависит, что резать.
- **Политика фасетов:** индексируемые комбинации — path + белый список параметров (одиночные `cityIds`, `specialtyIds`/категория, `lang`). Перестановки с `languageIds`, поиском по имени, множественными значениями → canonical на ближайшую базовую комбинацию или `noindex,follow`. Точка входа — `getCanonicalUrl` в `common/url-utils.ts` (сейчас self-canonical для ЛЮБОЙ комбинации параметров) и `robotsMeta` в `list-page.vue` (сейчас noindex только на пустых списках).
- **Sitemap:** пересмотреть, все ли классы комбинаций из `server/common/sitemap/sitemap.ts` заслуживают места (specialty×language — да, это лендинги «русскоговорящий стоматолог»; но проверить по GSC Performance, дают ли показы).
- **Гигиена:** 830 × 404, 77 × 5xx — выгрузить списки из GSC, починить. 341 «Google выбрал другой canonical» — посмотреть примеры (скорее всего lang-варианты или остатки ID-адресов).

### 4. Контент-слой из БД

В работе — PRD `prd/service-reference-content/` (115 карточек готово). Дальше:
- Авто-факты из собственной БД на фасетных страницах и страницах услуг: «N врачей/клиник, {услуга} от X €, средняя оценка Y» + FAQ. Цены — уникальный актив, которого нет ни у одного конкурента.
- Ручные тексты на топ-50 услуг по покрытию клиниками.

### 5. E-E-A-T

- About-страница с живым человеком (кто делает, зачем).
- Источники данных на видном месте: FZOCG-тарифы — официальные, это сильный аргумент; CInMED для лекарств.
- Контакты на своём домене (сейчас Telegram + email на стороннем домене).
- Приглушить сквозную связку с busticket4.me: сквозные взаимные ссылки сайтов одного владельца Google дисконтирует, паттерн похож на сеть сайтов. Оставить ссылку только на главной или nofollow.

### 6. Заработанные ссылки

Заработанных ссылок на docta.me ≈ 0 (972 из 1 081 — свой busticket4.me, остальное — унаследованное через 301 с купленных доменов клиник; редирект с чужого брошенного бренда передаёт мало доверия, хотя людей приводит — канал сохранить).

- Бейдж «профиль на docta.me» клиникам — у сотен клиник в базе есть сайты, docta.me уже ссылается на них с `utm_source=docta.me`, есть повод для контакта. ВАЖНО: продажа dofollow-ссылок выведена из биллинга (Google link spam policy) — бейдж должен быть бесплатным и добровольным.
- Русскоязычные сообщества экспатов в Черногории, локальные каталоги (hercegnovi.me уже ссылается).
- Реалистичная цель: 30–50 живых ссылающихся доменов за полгода — для ниши достаточно, конкуренты не богаче.

## Проверки в GSC перед стартом пунктов 1 и 3

1. Performance → фильтр по страницам с `specialtyIds` — сколько показов даёт весь класс фасетных страниц (базовая линия до перелинковки).
2. Performance → фильтр по `?lang=` — есть ли вообще показы у языковых версий (ru/en — целевой сегмент роста).
3. Выборка URL из «просканировано, но не проиндексировано» и «noindex» — состав определяет приоритеты пункта 3.
4. Инструмент проверки URL на паре фасетных перестановок — какой canonical выбрал сам Google.
