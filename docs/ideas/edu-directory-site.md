# Идея: каталог образовательных учреждений (edu-сайт)

> Статус: идея, не в работе. Анализ от 2026-06-12.
> Когда дойдёт до реализации — превратить в PRD в `prd/`.

## Суть

Отдельный сайт-каталог образовательных учреждений на основе кодовой базы docta.me:
университеты, школы, сады, доп. образование, спортивные секции, языковые курсы.
В docta.me добавляется образование врачей («Иван Иванов, образование: Белградский
университет») со ссылкой на страницу вуза на новом сайте.

Образование врачей — по всему региону (экс-Югославия + лениво СНГ), потому что врачи
в Черногории учились в Белграде, Сараево, Баня-Луке, реже — в самой Черногории.

## Ключевой вывод анализа

Внутри идеи — **два независимых продукта**, их нельзя связывать жёстко:

1. **Образование врачей в docta.me** — ценно само по себе (доверие пациента, контент
   профиля для SEO). Делается без зависимости от второго сайта.
2. **Edu-каталог** — отдельный продукт со своей аудиторией и экономикой.

## Три аудитории

| Аудитория | Что ищет | География |
| :-- | :-- | :-- |
| Родители-экспаты в MNE | школы, сады, секции, языковые курсы | гиперлокально (Черногория) |
| Абитуриенты | университеты | регионально (экс-YU) |
| Пациенты docta.me | проверка образования врача | медфакультеты региона |

Выигрышный приём тот же, что у docta.me: **фильтр по языку обучения**
(«школа с русским/английским в Подгорице») — этого нет у местных каталогов.

## Асимметричный охват — решение «всего региона» без взрыва скоупа

- **Региональный охват нужен только университетам, и их конечное малое число.**
  Медфакультетов в экс-Югославии ~18 (Подгорица, Белград, Нови-Сад, Ниш, Крагуевац,
  Сараево, Баня-Лука, Фоча, Тузла, Мостар, Загреб, Сплит, Риека, Скопье, Любляна…),
  плюс стоматология/фармация — 30–50 записей покрывают ~95% врачей.
- Школы/сады/секции — только Черногория (там аудитория).
- Вузы СНГ (Москва, Киев…) — лениво, как «заглушки» (название + город + страна,
  без полной страницы, noindex), по мере появления в анкетах врачей.

## Образование врачей: `doctor_education` (делать первым, в docta.me)

- `doctor_id`, `degree_type` (основное / специализация / субспециализация /
  магистратура / PhD / курсы), `institution_name_*` (6 локалей, как везде),
  `institution_id` (NULL — появится после запуска edu-сайта), `year_start`, `year_end`.
- **Специализация у врачей часто проходит в клиническом центре, а не в вузе**
  (KCCG, КЦ Сербии — KCCG уже есть в БД как клиника). Полиморфную ссылку
  «вуз или клиника» в MVP не делать: свободный текст обязателен всегда, ссылка опциональна.
- Название института хранить **денормализованно** в docta.me даже после появления
  `institution_id` — профиль врача не зависит от доступности второго сайта при SSR
  и переживёт смерть edu-проекта.

## Связка сайтов: главная ценность — обратная

- Прямая ссылка «вуз → страница вуза» пациенту даёт немного.
- Killer feature: на странице медфакультета блок
  **«Выпускники, практикующие в Черногории: N врачей»** со ссылками на профили
  docta.me. Уникальный контент, генерируется автоматически из данных docta.me,
  сразу решает проблему «тонких» страниц вузов на старте.
- SEO: перекрёстные ссылки между своими сайтами легитимны, пока редакционно уместны.
  Не включать ссылки из docta.me, пока страницы вузов — пустые заглушки.

## Код: форк, а не общий Nuxt layer

Сайты структурно изоморфны (institutions ≈ clinics, типы учреждений ≈ clinic_types,
программы с ценами ≈ clinic_medical_services, отзывы/карты/языки/города — один в один),
но общий слой для одного разработчика — ловушка: большой рефакторинг docta.me +
каждое изменение общего компонента проверять на двух сайтах. **Форк**; к общему слою
возвращаться, только если оба сайта проживут год и боль дублирования станет реальной.

Переиспользуется при форке: auth целиком, отзывы с модерацией и AI-summary,
импорт отзывов из Google Maps (для школ работает так же), list-page/list-card,
контакты, карты, биллинг (HIGHLIGHT/APPROVED — частные школы платят за продвижение
охотнее клиник).

Изменения в модели данных edu-сайта:

- `cities` + `country_id` (таблица `countries` с 6 локалями уже есть, переедет как есть);
- иерархия «университет → факультет» (`parent_id`) — врач учился на факультете;
  у школ иерархии нет;
- программы/курсы вместо услуг: возрастные группы для секций, язык обучения
  как первоклассный фильтр.

БД — отдельная схема на том же MySQL-сервере; docta.me при простановке
`institution_id` читает каталог напрямую (или через крохотный внутренний API),
но в рантайме от edu-сайта не зависит.

## Риски

- Главный риск не технический: **4 вертикали × 5 стран ≈ год работы** уровня docta.me,
  при том что в docta.me есть незакрытые хвосты (Stripe, ручная проверка кабинета).
- Источники данных: реестры школ министерства образования MNE, списки аккредитации
  вузов — опыт скрейпинга/OCR переносится.
- Отзывы о школах чувствительнее к диффамации (учителя, дети) — модерация уже есть.

## Приоритет вертикалей (по спросу экспатов)

1. Школы и сады Черногории.
2. Языковые курсы (вероятно самый частотный экспатский запрос — «курсы черногорского»).
3. Детские секции.
4. Университеты региона — маленький набор, делается быстро и сразу включает связку
   с врачами.
5. Потом: автошколы, музыкальные школы и пр. — каталог типов переживёт расширение
   без миграций.

## Порядок действий

1. `doctor_education` со свободным текстом в docta.me (без зависимостей, ценно сразу).
2. Форк репозитория под edu-сайт, мультистрановые города.
3. Университеты региона + блок выпускников — первая вертикаль (маленькая, уникальная,
   связывает сайты).
4. Школы/сады/курсы Черногории — основная контентная работа.
