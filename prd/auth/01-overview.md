## 1.1 Цель

Создать систему авторизации, начиная с **авторизации для админки** (email-based), затем добавить OAuth для обычных пользователей (Google, Telegram) с управлением ролями и правами доступа для редактирования клиник и профилей врачей.

## 1.2 Контекст

В текущей системе отсутствует авторизация. Все данные о врачах и клиниках статичны. 

**Приоритет #1:** Авторизация для администраторов
- Администраторы входят по email + password
- Аккаунты администраторов создаются и подтверждаются вручную в БД (нет открытой регистрации)
- Это закрытая система для управления контентом

**Приоритет #2:** OAuth для обычных пользователей
- Пользователи смогут создавать аккаунты через OAuth (Google, Telegram)
- Станет возможность для пользователей становиться врачами, создавать и управлять клиниками

## 1.3 Предположения и ограничения

- **Технологии:** Nuxt 3, MySQL, Nuxt Auth (или аналог для OAuth)
- **Админ авторизация:** Email + password, аккаунты создаются вручную в БД (нет регистрации)
- **OAuth провайдеры (для пользователей):** Google, Telegram (приоритет), возможность добавления других позже
- **Безопасность:** Все изменения данных требуют аутентификации, критичные операции требуют верификации
- **Локализация:** UI должен поддерживать все существующие языки (sr, sr_cyrl, ru, en, de, tr)
