# 1. Обзор

---

## 1.1 Цель

Создать полноценную систему авторизации пользователей с поддержкой OAuth (Google, Telegram), управлением ролями и правами доступа для редактирования клиник и профилей врачей.

## 1.2 Контекст

В текущей системе отсутствует авторизация. Все данные о врачах и клиниках статичны. Необходимо добавить возможность для пользователей создавать аккаунты, становиться врачами, создавать и управлять клиниками с системой подтверждений.

## 1.3 Предположения и ограничения

- **Технологии:** Nuxt 3, MySQL, Nuxt Auth (или аналог для OAuth)
- **OAuth провайдеры:** Google, Telegram (приоритет), возможность добавления других позже
- **Безопасность:** Все изменения данных требуют аутентификации, критичные операции требуют верификации
- **Локализация:** UI должен поддерживать все существующие языки (sr, sr_cyrl, ru, en, de, tr)
