## Цель

Создать админ панель для верификации клиник и управления системой.

## Зависимости

Все предыдущие итерации

## Задачи

1. Создать API endpoints для админских операций
2. Создать страницу админ панели
3. Создать UI для верификации клиник
4. Создать просмотр аудит логов

## Ключевые API Endpoints

- `GET /api/admin/clinics/pending` - список клиник на верификации
- `PUT /api/admin/clinics/[id]/verify` - верифицировать клинику
- `PUT /api/admin/clinics/[id]/reject` - отклонить клинику
- `GET /api/admin/audit-logs` - просмотр аудит логов

## Ключевые компоненты

- `pages/admin/index.vue` - главная страница админ панели с вкладками
- `components/admin/ClinicVerificationList.vue` - список клиник на проверке
- `components/admin/AuditLogTable.vue` - таблица аудит логов

## Критерии приемки

- [ ] AC-1: Страница /admin доступна только суперадминам
- [ ] AC-2: На вкладке "Верификация клиник" показываются все клиники со статусом pending_verification, draft, rejected
- [ ] AC-3: Клиники отсортированы: pending_verification сверху
- [ ] AC-4: Суперадмин может просмотреть детали клиники
- [ ] AC-5: Суперадмин может верифицировать клинику - статус меняется на 'published'
- [ ] AC-6: Суперадмин может отклонить клинику с указанием причины - статус меняется на 'rejected'
- [ ] AC-7: На вкладке "Аудит логи" показываются все записи из audit_logs
- [ ] AC-8: Логи отсортированы по дате (новые сверху)
- [ ] AC-9: Работает пагинация логов
- [ ] AC-10: После верификации/отклонения создается запись в audit_logs
