## Итерация 2: Геолокация и сортировка по расстоянию ✅

**Статус:** 🟢 Completed (2026-06-12)  
**Цель:** Определять город пользователя по IP и сортировать клиники по расстоянию.

**📄 [Детальное описание →](iteration-02-geolocation.md)**

### Ключевые фичи

🎯 Автоматическое определение города по IP (без запроса разрешения браузера)  
🎯 Сортировка клиник по расстоянию от пользователя (ближайшие первыми)  
🎯 Отображение расстояния на карточках клиник  
🎯 Компонент выбора города в UI  
🎯 Сохранение выбранного города (localStorage + БД)  
🎯 **НЕ фильтр** - геолокация влияет только на сортировку

### Задачи

1. Настроить IP Geolocation API
2. Создать функцию расчета расстояния (Haversine)
3. Обновить API списка клиник для сортировки по расстоянию
4. Создать composable `useUserLocation`
5. Создать компонент `LocationSelector`
6. Интегрировать в страницу клиник
7. Обновить `ClinicCard` для отображения расстояния
8. Добавить сохранение в БД для авторизованных пользователей

### Критерии приемки

- [x] Город определяется автоматически по IP при первом заходе
- [x] Список клиник сортируется по расстоянию
- [x] Расстояние отображается на карточках (км/м)
- [x] Пользователь может вручную изменить город
- [x] Выбранный город сохраняется (localStorage + БД — ⚠️ ждёт миграцию 004)
- [x] Геолокация не скрывает клиники из других городов
- [x] При ошибке используется fallback (Podgorica)
- [x] Работает на всех 6 языках
