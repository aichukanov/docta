## Что было до итерации

На `/clinics` (и других listing-страницах) уже существовала **боковая карта**
(`components/clinic-services-map.vue` внутри `components/list-page.vue`):
sticky-сайдбар справа на десктопе, блок 450px внизу на мобильном. Она
показывала **все** клиники без учёта фильтров и без кластеризации.

## Что сделано

- **Переключатель «Список | Карта»** рядом с заголовком `/clinics`
  (слот `header-actions` в `list-page.vue`). Состояние — в URL
  (`?view=map`), переживает навигацию и обновление страницы.
- **Режим карты:** список и пагинация скрыты, карта занимает основную
  площадь; слева остаётся колонка с заголовком и фильтрами (десктоп).
  На **мобильном — полноэкранный режим** (fixed под шапкой) с плавающей
  кнопкой «К списку».
- **Кластеризация** — leaflet.markercluster (1.5.3, грузится по требованию с
  unpkg, как сам Leaflet). Поддержка добавлена в `composables/use-leaflet.ts`
  (`initializeMap(..., { cluster: true })`). Иконки кластеров и маркеров — на
  design-токенах.
- **Попап маркера** — переиспользован `MapClinicPopup`: название (ссылка на
  страницу клиники), адрес с кнопкой маршрута, языки, контакты.
- **Карта показывает клиники по текущим фильтрам** — отдельный запрос
  `/api/clinics/list` без пагинации (client-only). Бонус: боковая карта
  режима списка теперь тоже отражает фильтры (раньше показывала все клиники).
- Новый компонент: `components/clinics-map-view.vue`. Маркеры — статичный
  HTML (`divIcon`), не Teleport: markercluster пересоздаёт DOM маркеров, и
  телепорт-цели внутри них пропадают (этим режим отличается от
  `clinic-services-map.vue`).
