### ✅ Итерация 1: Публичная страница клиники

Полностью функциональная публичная страница с:

**Основная информация:**

- Название, адрес, координаты
- Языки сопровождения
- Контакты (телефон, email, мессенджеры)
- Описание клиники
- Approved badge (✓)
- Кнопки "Показать на карте" и "Маршрут"

**Контент:**

- Врачи (сгруппированные по специальностям)
- Медицинские услуги (сгруппированные по категориям) с ценами
- Анализы (сгруппированные по категориям) с ценами
- Лекарства с ценами

**Техническое:**

- SEO и Schema.org
- Локализация (6 языков)
- Адаптивный дизайн

**Файлы:**

- `pages/clinics/[clinicId]/index.vue`
- `components/clinic/header.vue`
- `components/clinic/summary.vue`
- `components/clinic/categorized-section.vue`
- `server/api/clinics/details.post.ts`

## Что реализовано в итерациях 2–6 (2026-06-11)

Вместо отдельных страниц `/profile/clinics/create|edit` кабинет встроен
в таб «Мои клиники» профиля (`components/profile/tab/clinics.vue`) —
по образцу таба врача.

### Итерация 2: Создание клиники ✅

- `clinics.created_by` (FK auth_users) + `status` — миграция `004-clinic-cabinet.sql`
- `POST /api/clinics/my-create` — клиника создаётся черновиком
- У пользователя может быть несколько клиник (`GET /api/clinics/my-list`)

### Итерация 3: Редактирование профиля ✅

- Форма `components/profile/ClinicEditForm.vue` + `POST /api/clinics/my-update`
- Загрузка логотипа (категория `clinics` в /api/upload/image)
- Map picker координат: `ClinicMapPicker.vue` (Leaflet, `onMapClick` в use-leaflet)
- Слаг следует за названием, старый сохраняется в slug_redirects

### Итерация 4: Управление контентом ✅ (скорректирован скоуп)

- Кабинет управляет профильными данными: контакты, языки сопровождения,
  типы клиники, график работы (`ClinicWorkingHoursEditor.vue`), описание
- Врачи/услуги/анализы/лекарства клиники остаются в админке
  (пользовательское управление — кандидат в отдельный PRD)

### Итерация 5: Markdown & Preview ✅

- Переиспользованы `markdown-editor.client.vue` (редактор с превью)
  и `marked-content.vue` (рендер) через `LocalizedFieldEditor type="markdown"`
- Описание на 6 локалях

### Итерация 6: Draft/Published режимы ✅

- `status ENUM('draft','pending_verification','published','rejected')`
- Пользовательский флоу: draft ↔ published (`POST /api/clinics/my-set-status`);
  публикация требует название+адрес+точку на карте
- Draft виден только владельцу/админу (остальным 404, noindex), исключён
  из списков, селектов и sitemap
- `pending_verification`/`rejected` зарезервированы под PRD clinic-verification
