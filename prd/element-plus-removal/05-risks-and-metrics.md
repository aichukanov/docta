## Риски

| # | Риск | Вероятность | Импакт | Смягчение |
|---|---|---|---|---|
| R-1 | A11y-combobox (AppSelect) окажется сильно дороже оценки: клавиатура, фокус, скрин-ридеры | Средняя | Высокий | Скоуп ограничен фактическим использованием (без виртуализации, групп, тегов-коллапса). Жёсткий fallback: Reka UI Listbox/Combobox как headless-основа, стили свои. Решение — не позже конца итерации 2 |
| R-2 | A11y-регрессии относительно закрытого WCAG-аудита 2026-06 | Средняя | Высокий | NFR-1 как чек-лист приёмки каждого компонента; точечный повторный аудит новых компонентов в итерации 5 |
| R-3 | Регрессии в денежных/критичных потоках (auth, billing/Stripe, верификация отзывов) | Низкая | Высокий | Эти зоны мигрируются в итерации 4 отдельным заходом, после обкатки кита на публичном контуре; e2e-прогон + ручная проверка чек-листом итерации |
| R-4 | Мерцание гидрации / SSR-ошибки новых компонентов на публичных страницах | Средняя | Средний | NFR-2; селекты рендерят триггер сервером, дроп — только клиентом; проверка `npm run build && npm run preview`, не только dev |
| R-5 | Миграция растянется, проект надолго зависнет с двумя системами | Высокая | Низкий | Это штатный режим, а не провал: обе системы на одних токенах, разнобоя нет. Контуры самодостаточны — ценность фиксируется после каждой итерации, паузы безопасны |
| R-6 | «Расползание» кита: желание заодно отрефакторить дизайн | Средняя | Средний | Жёсткое правило из 01-overview: визуальный паритет по умолчанию, редизайн — отдельные задачи вне PRD |
| R-7 | Поломка обновлением до удаления: транзитивный `@element-plus/icons-vue` исчезнет/сменит версию при обновлении element-plus | Низкая | Средний | Иконки мигрируются в итерации 1 первыми — риск снимается в начале, а не в конце |
| R-8 | Альтернативная стоимость: P2-техдолг против продуктовых фич (фильтры лекарств и пр.) | — | — | Решение об очерёдности за владельцем проекта; PRD спроектирован так, что миграцию можно прерывать после любой итерации |

## Открытые вопросы

| # | Вопрос | Когда решить |
|---|---|---|
| Q-1 | Свой combobox vs Reka UI для AppSelect | Конец итерации 2 (по факту трудозатрат) |
| Q-2 | Мигрировать ли админку (итерация 5) или заморозить EP в ней надолго | После итерации 4. Если заморозка — пакет остаётся, мост `--el-*` остаётся, цель «0 зависимостей» не достигается; бандл-выгоды публичного контура при этом сохраняются полностью (admin — `ssr:false`, отдельные чанки) |
| Q-3 | Нужна ли dev-витрина компонентов (`/dev/ui`) | Итерация 1, по желанию — сильно ускоряет визуальную проверку |
| Q-4 | Заменить набор иконок (EP-копии → свой/lucide-стиль) | Вне scope; после миграции это локальная правка `ui/icons/` |
