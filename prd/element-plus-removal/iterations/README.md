## Принцип нарезки

Миграция идёт **контурами** от максимальной пользы к минимальной:
публичные SSR-страницы → auth/профиль/кабинет → админка. Каждая итерация
самодостаточна: после неё проект в стабильном состоянии, паузу можно держать
сколь угодно долго (кит и EP сидят на одних токенах, визуального разнобоя нет).

Каждая итерация закрывается тройным гейтом:

1. **Grep-гейт контура** — в его файлах 0 вхождений `<el-`, `\bEl[A-Z]`, `v-loading`, `@element-plus`;
2. `npm run typecheck` зелёный;
3. e2e (`npm run test:e2e`) зелёный + ручной чек-лист итерации.
