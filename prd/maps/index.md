## Контекст

Карты построены на одном composable `composables/use-leaflet.ts` (Leaflet 1.9.4 с CDN, опционально leaflet.markercluster). Потребители:

1. **Полноэкранный режим «Карта» на /clinics** — `components/clinics-map-view.vue`: кластеризация, статичный HTML-маркер, данные — отдельный запрос `/api/clinics/list` без пагинации (`key: 'clinics-map-list'`).
2. **Боковая карта listing-страниц** — `components/clinic-services-map.vue` из `components/list-page.vue`: маркеры через Teleport в пустые `divIcon`-контейнеры, без кластеризации.
3. **Детальные страницы** (услуга/анализ/лекарство) — `clinic-services-map.vue`, клиники из стора по полному `clinicIds` из details-эндпоинта.

### Уже исправлено (2026-06-12)

- listing-эндпоинты (services/labtests/medications) отдают **полный** `clinicIds` (карточка сама режет до `LIST_CARD_MAX_CLINICS`, `clinicPrices` по-прежнему обрезаны на бэкенде);
- guard в `addMarker`: клиника без координат пропускается, а не роняет добавление остальных маркеров;
- `clinic-services-map.vue` фильтрует клиники без координат;
- авто-`fitBounds` по видимым маркерам (`autoFit`, включён когда город не выбран — иначе центрирование по городу из `list-page.vue`).

## Итерация 1: Кластеризация боковой карты

**Проблема:** на боковой карте (`clinic-services-map.vue`) нет кластеризации — близко стоящие клиники перекрывают друг друга, кликабелен только верхний маркер.

**Сложность:** Teleport-маркеры несовместимы с markercluster — плагин пересоздаёт DOM маркеров при сборке/разборке кластеров, и Teleport-цели пропадают (см. комментарий в `AddMarkerOptions.html` в `use-leaflet.ts`).

**Решение:** перевести маркеры на статичный HTML (как уже сделано в `clinics-map-view.vue`): бейдж с числом услуг рендерить строкой в `html` опции `addMarker`, клик вешать через `onClick`. После этого включить `initializeMap(..., { cluster: true })`.

**Заодно закрывает:** невидимые пустые маркеры. Сейчас в режиме услуг маркеры создаются для **всех** клиник стора, а бейджи телепортируются только в клиники с услугами — пустые прозрачные `divIcon` 40×40 могут перехватывать клики над видимыми маркерами. При переходе на статичный HTML создавать маркеры только для `clinicsWithServices` (синхронизацию завязать и на `services`, не только на `clinics`).

## Итерация 2: Полнота карты на listing-страницах услуг

**Проблема:** боковая карта на /services, /labtests, /medications, /doctors отражает только текущую страницу списка (20 элементов). Клиники, чьи услуги не попали на страницу, на карте отсутствуют.

**Решение:** по аналогии с /clinics — отдельный запрос без пагинации по текущим фильтрам (на /clinics это `useFetch` с `key: 'clinics-map-list'`, `server: false, lazy: true`). Для услуг эндпоинту нужна лёгкая агрегация «клиника → число услуг по фильтру» (не тянуть полные элементы без пагинации — дорого).

**Открытый вопрос:** UX — должна ли карта отражать список (текущее поведение) или весь фильтр. Решили двигаться к «весь фильтр», но бейдж числа услуг тогда означает «всего по фильтру», а не «на этой странице».

## Итерация 3: Технические риски и полировка

### 3.1. `group_concat_max_len`

Дефолт MySQL — 1024 байта, нигде не поднимается. `clinicPricesData` (формат `clinicId:price:priceMin:priceMax:code`) усечётся примерно после 40–70 клиник на услугу → битые id, потерянные клиники и цены. На 2026-06-12 максимум ~259 байт, но импорты прайс-листов быстро приближают к лимиту.

**Решение:** поднять на уровне сессии/пула в `server/common/db-mysql.ts` (например, `SET SESSION group_concat_max_len = 65535` в `pool.on('connection')` или через `connectionAttributes`/init-запрос пула).

### 3.2. Визуальное смещение маркеров на ~20px

`divIcon` уже центрирован через `iconAnchor: [20, 20]`, но CSS маркеров (`.clinic-service-marker`, `.clinics-map-marker`, `.map-cluster-marker`) дополнительно сдвигает их `transform: translate(-50%, -50%)` — двойная компенсация, маркер визуально на 20px выше-левее реальной точки. Попап подогнан под это смещением `offset: [-20, -30]` в `use-leaflet.ts`.

**Решение:** убрать `translate(-50%, -50%)` из CSS маркеров (оставить только в `:hover`-масштабировании, пересчитав), вернуть попапу естественный offset. Править синхронно во всех трёх местах + проверить позиционирование попапа.

### 3.3. Прочее

- Дубли координат в данных пока отсутствуют (проверено 2026-06-12: 131 клиника, 0 без координат, 0 точных дублей) — при появлении дублей без кластеризации маркеры полностью перекроются.
- Leaflet грузится с unpkg.com — при недоступности CDN карта не работает; рассмотреть self-host через npm-пакет.
