## Фаза 1 — геолокация и сортировка по расстоянию (итерация 2)

1. `server/api/geo/detect-location.get.ts`: определение города по IP через ipapi.co
   (таймаут 3с), фолбэк Podgorica (42.4304, 19.2594) для локальных IP и ошибок;
   сматчить ответ с таблицей `cities` в БД. Кэшируй результат по IP на сервере
   (in-memory достаточно), чтобы не сжечь лимит 1000 req/день.
2. Composable `useUserLocation`: запрашивает detect-location, хранит выбор в
   localStorage (TTL 30 дней), для авторизованных — сохранение в БД (поле к users,
   см. правило про миграции ниже).
3. Сортировка по расстоянию в `server/api/clinics/list.ts`: Haversine в ORDER BY,
   когда координаты пользователя известны. Это сортировка, НЕ фильтр — все клиники
   остаются видимыми. Клиники без координат — в конец.
4. Компонент `LocationSelector` рядом с фильтрами: показывает определённый город,
   позволяет сменить (список из `cities`), сбросить на «автоматически».
5. Расстояние на карточке клиники («2,3 км») — только когда локация известна;
   формат числа локализуй.
6. SEO-вывод (SSR без знания локации) не должен измениться — сортировка по
   расстоянию применяется только когда локация реально определена.

## Фаза 2 — остаток расширенных фильтров (итерация 3)

Сверь `iterations/iteration-03-advanced-filters.md` с уже существующими фильтрами.
Типично недостающее: фильтр по рейтингу (данные отзывов в БД уже ЕСТЬ — внутренние
и импортированные внешние), по специализации/услугам. Реализуй недостающее по
паттерну существующих: компонент в `components/filter/`, неймспейс в filters store,
синк с URL query, учёт фильтра в заголовке страницы (см. pageTitle в
`pages/clinics/index.vue`). Если какой-то фильтр считаешь нецелесообразным —
не реализуй, но зафиксируй причину в PROGRESS.md.

## Фаза 3 — карта клиник (итерация 4)

Переключатель список/карта на `/clinics` (состояние в URL query, не теряется при
навигации). Карта на Leaflet по образцу `clinic-services-map.vue`, кластеризация
маркеров, попап с названием/адресом/ссылкой на клинику. Карта показывает клиники
по текущим фильтрам. На мобильном — полноэкранный режим карты.

## Конвенции проекта (обязательно)

- SQL-миграции НЕ применять: файл в `server/sql/migrations/`, выдай пользователю
  готовую команду mysql, после его подтверждения отметь в `server/sql/migrations/APPLIED.md`.
- i18n: 6 локалей (en, ru, sr, sr-cyrl, de, tr), сербский — иекавица, sr-cyrl
  зеркалит sr кириллицей. Локальные сообщения через useI18n scope-паттерн как в
  существующих страницах. Названия городов уже локализованы — переиспользуй.
- Дизайн: только токены design-tokens.css, никаких hex; листинги — типографика
  по образцу существующих карточек. AppBreadcrumbs, не el-breadcrumb.
- `npm run typecheck` зелёный после каждой фазы.
- НЕ запускай `nuxt build`, пока работает dev-сервер пользователя.
- НЕ коммить — пользователь коммитит сам.
- Не трогай файлы вне своей зоны: `composables/use-analytics.ts` и трекинг-события
  делает другое окно.
