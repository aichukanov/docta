## Что сделать

1. **Итерация 3 — верификация отзывов**: загрузка подтверждения (чек/назначение)
   к отзыву через существующий `composables/use-image-upload.ts`; статус
   верификации в БД; бейдж «верифицированный отзыв» в `components/review/item.vue`;
   проверка подтверждения админом (см. п.2 — сделай частью модерационной очереди).
   Файлы подтверждений НЕ публичны — доступ только админу и автору.
2. **Итерация 6 — модерация**: очередь модерации в админке (новые отзывы,
   отзывы с жалобами, заявки на верификацию): approve/reject с причиной,
   фильтры по статусу/типу. Расширь существующие admin-API, права — по образцу
   текущих admin-endpoint'ов. Реши по спеке, нужна ли пре-модерация (отзыв виден
   после approve) или пост-модерация — зафиксируй решение в PROGRESS.md.
3. **Итерация 5 — AI summary** (последней, она наименее важна): суммаризация
   отзывов сущности (врач/клиника) при количестве >= порога (возьми из спеки,
   иначе 5); кэш в БД с датой генерации, регенерация при появлении N новых
   отзывов; генерация на основном языке контента + перевод по существующему
   workflow локализаций ИЛИ генерация на лету для текущей локали — выбери
   проще и дешевле, обоснуй в PROGRESS.md. Блок на странице отзывов с пометкой
   «AI-обзор». Генерацию запускай асинхронно (не блокируя запрос пользователя).

## Конвенции проекта (обязательно)

- SQL-миграции НЕ применять: файл в `server/sql/migrations/` + команда mysql,
  после подтверждения пользователя — отметка в `APPLIED.md`.
- i18n: 6 локалей (en, ru, sr, sr-cyrl, de, tr), иекавица.
- Только дизайн-токены; AppBreadcrumbs; переиспользуй review-компоненты.
- `npm run typecheck` зелёный. НЕ запускай `nuxt build` при работающем dev-сервере.
- НЕ коммить — пользователь коммитит сам.
- Зона: `server/api/reviews/**`, `components/review/**`, `components/admin/review*`,
  `pages/admin/**`. Карточки врачей/клиник и rating-stars не трогать (зона окна 2).

## Definition of Done

- Верификация: загрузка → очередь → approve → бейдж на отзыве.
- Модерация: полный цикл approve/reject из админки.
- AI summary: появляется при достижении порога, без ключа — выключен.
- typecheck зелёный, тексты на 6 локалях.
- `prd/reviews/PROGRESS.md` и итерации обновлены (итерации 1, 2, 4 — done ранее).
- Сводка: файлы, миграции, env-ключи, сценарии ручной проверки.
