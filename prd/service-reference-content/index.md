# Справочный контент на страницах услуг и анализов

## Проблема

Страницы `pages/services/[serviceSlug]/index.vue` и `pages/labtests/[labTestSlug]/index.vue` содержат только вкладки «Клиники / Тариф ФЗОЦГ / Карта» — текстового контента нет. Это часть причины thin-content потолка сайта (см. [[seo-audit-2026-07]], пункт 4 плана).

## Формат справки (единый для всех страниц)

Пять полей, без воды, таблицей:

| Поле | Смысл | Ограничение по формулировке |
|---|---|---|
| **Что это** (`what`) | 1 фраза, что за анализ/процедура | Без диагностических ярлыков |
| **Как проводится** (`how`) | Что физически происходит, сколько занимает | Только процедура, не результат |
| **Когда назначают** (`when`) | Типичные показания | Общие ситуации, не «у вас есть X» |
| **Подготовка** (`prep`) | Условия перед процедурой | Конкретно и проверяемо (натощак, часы) |
| **О чём может говорить отклонение** (`abnormal`) | Направление, не диагноз | Обязательно «может быть связано с» + отсылка к врачу |

Плюс единый дисклеймер на каждой странице (не дублируется в данных — общая i18n-строка):

> Информация носит справочный характер и не заменяет консультацию врача. Референсные значения и трактовка результата зависят от лаборатории/клиники и индивидуальных особенностей пациента.

### YMYL-правила

- Никаких числовых референсных диапазонов (разные у лабораторий/приборов).
- Никаких формулировок «повышение X означает болезнь Y» — только «может быть связано с» + отсылка к врачу.
- Никаких рекомендаций по лечению/дозировкам.
- Онкомаркеры/ВИЧ/гепатиты/сифилис — только нейтральный факт-контент (что это, зачем берут), без интерпретации риска и без стигматизирующих формулировок.
- Пренатальный скрининг с расчётом риска (Double/Triple test, PAPP-A) — вне охвата, слишком сложна нюансная интерпретация для нейтрального текста.
- Тесты на наркотики — вне охвата (не медицинский чекап по сути, юридический контекст).

## Хранение (текущее решение)

Пока — JSON-файлы, переводы внутри. Позже отдельной задачей — миграция в SQL (новая таблица, аналогично `medical_service_tariffs`: `entity_type` + `entity_id`/`slug` + поля по локалям, следуя конвенции `name_en/name_sr/name_sr_cyrl/name_ru/name_de/name_tr` из `lab_tests`/`medical_services`).

Файлы:
- `data/entity-reference/lab-tests.json`
- `data/entity-reference/medical-services.json`
- `data/entity-reference/README.md` — схема

Схема на 1 элемент:

```json
{
  "slug": "complete-urinalysis",
  "entity_type": "lab_test",
  "translations": {
    "ru": {
      "what": "...",
      "how": "...",
      "when": "...",
      "prep": "...",
      "abnormal": "..."
    }
  }
}
```

Локали `en/sr/sr_cyrl/de/tr` добавляются в `translations` по мере перевода — не блокируют черновик на русском.

## Не решено
