# 1. Обзор

## 1.1 Цель

Создать детальный профиль пользователя, где можно:

- Управлять базовой информацией (имя, email, фото)
- Управлять OAuth привязками (добавлять/удалять Google, Telegram)
- Настраивать приватность профиля
- Маскировать email для публичного отображения (в комментариях)

## 1.2 Контекст

После реализации базовой авторизации (auth PRD) пользователи могут войти через OAuth и видеть блок пользователя в хэдере. Однако у них нет детальной страницы профиля, где можно было бы:

- Посмотреть все привязанные OAuth аккаунты
- Добавить или удалить OAuth провайдера
- Настроить, что показывать публично (если будут комментарии)

Этот PRD создает полноценный профиль пользователя.

## 1.3 Предположения

1. **auth PRD завершен** - базовая авторизация через OAuth работает
2. **Пользователи не имеют username** - используем email как идентификатор
3. **Маскировка email** - для анонимности в комментариях (x**\***@g\***_._**)
4. **Публичный профиль минимальный** - пока только для будущих комментариев

## 1.4 Ограничения

1. **Нет email/password авторизации** - будет добавлено позже в auth PRD
2. **Нет смены email** - требует верификации по email (будет позже)
3. **Нет смены пароля** - требует email/password auth
4. **Нет аватара пользователя** - используется фото из OAuth

## 1.5 Целевая аудитория

- **Пользователи** - обычные пользователи, оставляющие комментарии
- **Будущие врачи** - пользователи, которые позже создадут профиль врача
- **Будущие админы клиник** - пользователи, которые будут управлять клиниками
