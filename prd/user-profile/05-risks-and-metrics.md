## 5.1 Риски

| Риск                                                      | Вероятность | Влияние     | Mitigation                                                      |
| --------------------------------------------------------- | ----------- | ----------- | --------------------------------------------------------------- |
| Пользователь случайно удалит все OAuth и потеряет доступ  | Средняя     | Критическое | Не позволять удалять последний OAuth, показывать предупреждение |
| Маскировка email не обеспечивает достаточной анонимности  | Низкая      | Среднее     | Предоставить опцию "Приватный профиль" (полностью анонимный)    |
| Пользователи не понимают разницу публичный/приватный      | Средняя     | Низкое      | Добавить подсказки и кнопку "Предпросмотр"                      |
| Производительность при большом количестве OAuth аккаунтов | Низкая      | Низкое      | Индексы на oauth_accounts, limit 10 привязок на пользователя    |

## 5.2 Метрики успеха

### Adoption

- **% пользователей, посетивших /profile** - цель: 60% в первый месяц
- **% пользователей, добавивших 2+ OAuth** - цель: 15%
- **% пользователей, изменивших display_name** - цель: 30%

### Engagement

- **Среднее количество OAuth на пользователя** - baseline: 1.0, цель: 1.3
- **% пользователей, использующих приватный режим** - baseline: 0%, ожидаемое: 20-30%

### Качество

- **Успешность добавления OAuth** - цель: >95%
- **Успешность удаления OAuth** - цель: >99%
- **Время загрузки /profile** - цель: <300ms (p95)

### Проблемы

- **% попыток удалить последний OAuth** - мониторинг, чтобы понять частоту
- **Количество обращений в поддержку по профилю** - цель: <5 в месяц

## 5.3 Открытые вопросы

1. **Q:** Нужно ли позволять пользователям загружать собственный аватар?

   - **A:** Пока нет, используем фото из OAuth. Может быть добавлено позже.

2. **Q:** Что делать, если у пользователя нет фото ни в одном OAuth?

   - **A:** Показывать первую букву имени в круглом placeholder (как в auth PRD).

3. **Q:** Нужен ли публичный профиль пользователя (отдельная страница `/users/[id]`)?

   - **A:** Пока нет. Публичный профиль используется только для отображения в комментариях (маскированный email).

4. **Q:** Как часто пользователи должны обновлять фото из OAuth?

   - **A:** При каждом входе обновляем `photo_url` из OAuth провайдера.

5. **Q:** Нужна ли история изменений профиля?
   - **A:** Пока нет. Может быть добавлено в audit_logs позже для важных изменений.

## 5.4 Зависимости от других PRD

- **auth** - базовая авторизация (OAuth flow)
- **comments** (будущий PRD) - использование маскированного email в комментариях
